# ClickUp Setup Guide

Use this guide to create the Rediscovering Faith capstone workspace structure in ClickUp.

## Recommended Structure

Workspace:

- Your existing ClickUp workspace

Space:

- Capstone Projects

Folder:

- Rediscovering Faith

Lists:

- Sprint 1 - Foundation & Core Experience
- Sprint 2 - Community & Engagement
- Sprint 3 - Pastoral & Church Tools
- Sprint 4 - Admin, Groups & Deployment

Statuses:

- Backlog
- Ready
- In Progress
- Review
- Done

## Custom Fields

Recommended custom fields:

- Sprint
- Feature Area
- GitHub Branch
- GitHub Commit
- Demo Notes

## Import Steps

1. Open ClickUp.
2. Create or open the Rediscovering Faith location.
3. Create the list named `Sprint 1 - Foundation & Core Experience`.
4. Import `docs/clickup/sprint-1-clickup-import.csv`.
5. Map columns as closely as ClickUp allows:
   - `Task Name` to task name
   - `Description` to description
   - `Status` to status
   - `Priority` to priority
   - `Due Date` to due date
   - `Tags` to tags
   - `Feature Area`, `Sprint`, and GitHub fields to custom fields
6. Confirm the imported tasks appear in the Sprint 1 list.
7. Attach the GitHub repo URL to the ClickUp project or list description:
   - `https://github.com/evan-ryder1/rediscovering-faith`

## Working Rhythm

Daily:

- Move tasks into `In Progress` only when actively working on them.
- Add the related branch or commit after meaningful progress.
- Keep task comments short but specific.

End of Sprint 1:

- Move finished tasks to `Done`.
- Keep unfinished tasks in `Backlog` or move them to Sprint 2.
- Add a Sprint 1 recap to the repo.
